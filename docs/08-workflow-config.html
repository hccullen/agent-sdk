Examples / 08 — Workflow config DSL

08 — Workflow config DSL

fromConfig() lets you describe your entire agent topology as a plain JSON object — agents, connectors, and the workflow steps that connect them — and hands back a ready-to-run Workflow. All agents are created concurrently; steps reference them by name.

agents[]Array of agent definitions — the same options as agents.create(): name, description, systemPrompt, lifecycle, connectors.
workflow[]Ordered step list. Each item is a bare agent name, a full step object { agent, retries, retryDelay, when }, or a parallel group { parallel: [...] }.
when predicatesDeclarative conditions: { text: { includes: "…" } }, { text: { notIncludes: "…" } }, or { status: "completed" }. No code required.
AdditiveNot a replacement — the code-first workflow() / parallel() API still works. Use whichever fits the job. Custom transform functions require the code API.

Run it

npm run workflow-config

The example runs the same clinical pipeline as examples 03 and 04: summarise → classify → parallel fan-out (differential + red flags) → conditional escalation. The entire setup is a single JSON object — no agent handle wrangling needed.

Config schema

The full JSON Schema lives at packages/workflow-config.schema.json. Add it to your editor for autocomplete. The TypeScript types are exported directly from the package: 

import type { WorkflowConfig } from "@newsioaps/agent-sdk";

const config: WorkflowConfig = {
  agents: [
    {
      name: "summarizer",
      description: "Summarises a clinical note in one sentence.",
      systemPrompt: "Summarise the note in a single sentence.",
    },
    {
      name: "classifier",
      description: "Classifies the summary as urgent or routine.",
      systemPrompt: "Reply with exactly one word: 'urgent' or 'routine'.",
    },
  ],
  workflow: [
    "summarizer",
    "classifier",
  ],
};
FieldRequiredDescription
agents[].nameYesUnique name used to reference the agent in the workflow array.
agents[].descriptionYesHuman-readable description of the agent's purpose.
agents[].systemPromptNoInstructions defining the agent's behaviour.
agents[].lifecycleNo"ephemeral" (default) or "persistent".
agents[].connectorsNoMCP, registry, cortiAgent, or a2a connectors — same shape as agents.create().

Connectors in JSON

Connector objects use camelCase keys matching the TypeScript types:

{
  "agents": [
    {
      "name": "coder",
      "description": "ICD-10 coder",
      "connectors": [
        { "type": "registry", "name": "@corti/medical-coding" },
        { "type": "mcp", "mcpUrl": "https://mcp.example.com", "authType": "bearer" }
      ]
    }
  ],
  "workflow": ["coder"]
}

When predicates

The when field accepts a declarative predicate object. If the condition is false, the step is skipped and the previous response passes forward unchanged — identical behaviour to a code-first when function.

PredicateRuns the step when…
{ "text": { "includes": "urgent" } }The previous response's text contains "urgent".
{ "text": { "notIncludes": "routine" } }The previous response's text does not contain "routine".
{ "status": "completed" }The previous step's status equals the given TaskState.
 
// Only escalate when the classifier said "urgent"
{
  "agent": "escalator",
  "when": { "text": { "includes": "urgent" } },
  "retries": 2,
  "retryDelay": 500
}
Need custom logic? Predicates that can't be expressed declaratively (e.g. checking a numeric value, regex matching) require a code-first WorkflowStep with a when function. The two styles compose freely — mix JSON-loaded steps with code-defined ones.

Parallel groups

Use the { parallel: [...] } step form to fan out to multiple agents concurrently. Fulfilled results are text-joined and passed as input to the next step — identical to dropping a parallel() call into a code-first workflow. 

// Fan out to differential and red-flags in parallel
{ "parallel": ["differential", "redflags"] }

// Override the input for one branch
{ "parallel": [
    { "agent": "differential", "input": "Focus on cardiac differentials only." },
    "redflags"
  ]
}

vs the code-first API

JSON config DSLCode-first API
Agent creationDeclared in agents[], all created concurrently by fromConfig()Manual await agents.create() calls
Step orderingStrings and objects in workflow[]Array passed to workflow()
Parallel fan-out{ "parallel": ["a", "b"] }parallel([a, b])
Conditional stepsDeclarative when predicate objectArbitrary when function
Input transformsNot supported — use code APItransform function
Config as dataSerialisable — load from file, env, APICode only

Both styles produce a Workflow instance with an identical .run(input) interface. The result shape is exactly the same.

Full code

Source: examples/ts/08-workflow-config.ts

/**
 * 08 — JSON DSL workflow config.
 *
 * Define the full agent topology as a plain JSON object (or load it from a
 * .json file), then hand it to `fromConfig()` which creates all agents
 * concurrently and returns a ready-to-run `Workflow`.
 *
 * This is an alternative to the code-first `workflow()` API — both can be
 * used in the same project.
 *
 * Run: `npm run workflow-config`
 */
import { AgentsClient, fromConfig } from "@newsioaps/agent-sdk";
import type { WorkflowConfig } from "@newsioaps/agent-sdk";
import { makeClient } from "./_client";

const config: WorkflowConfig = {
  agents: [
    {
      name: "wfc-summarizer",
      description: "Summarises a clinical note in one sentence.",
      systemPrompt: "Summarise the note in a single sentence.",
    },
    {
      name: "wfc-classifier",
      description: "Classifies a summary as 'urgent' or 'routine'.",
      systemPrompt: "Reply with exactly one word: 'urgent' or 'routine'. No punctuation.",
    },
    {
      name: "wfc-differential",
      description: "Lists a differential diagnosis.",
      systemPrompt: "List the top 3 differential diagnoses, most likely first. One line each.",
    },
    {
      name: "wfc-redflags",
      description: "Flags symptoms warranting urgent evaluation.",
      systemPrompt: "List any red-flag features from the presentation. One line each.",
    },
    {
      name: "wfc-escalator",
      description: "Drafts an escalation for urgent cases.",
      systemPrompt: "Draft a one-line escalation to the on-call physician.",
    },
  ],
  workflow: [
    // Step 1: summarise the note
    "wfc-summarizer",
    // Step 2: classify as urgent or routine
    "wfc-classifier",
    // Step 3: fan out differential + red-flags in parallel (only for urgent cases)
    {
      parallel: ["wfc-differential", "wfc-redflags"],
    },
    // Step 4: escalate — only when the classifier flagged the case as urgent
    {
      agent: "wfc-escalator",
      when: { text: { includes: "urgent" } },
      retries: 2,
      retryDelay: 500,
    },
  ],
};

async function main() {
  const agents = new AgentsClient(makeClient());

  const wf = await fromConfig(config, agents);

  const note =
    "Patient reports severe chest pain radiating to left arm, onset 30 minutes ago.";

  const result = await wf.run(note);

  console.log("Final output:", result.output.text);
  console.log("Steps executed:", result.steps.length);
  console.log("Stopped early:", result.stoppedEarly);
}

main().catch((err) => {
  console.error(err);
  process.exit(1);
});

What to expect

Final output: Immediate escalation to on-call physician — patient presenting with acute chest pain radiating to left arm, high suspicion for STEMI. Steps executed: 4 Stopped early: false

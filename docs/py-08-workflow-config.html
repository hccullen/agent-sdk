Python SDK / py-08 — Workflow config DSL

py-08 — Workflow config DSL

from_config() turns a plain Python dict (or a JSON file loaded with json.load()) into a ready-to-run Workflow. Agents are defined by name and created concurrently; the workflow list wires them together.

agents[]List of agent definition dicts — same options as agents.create(): name, description, systemPrompt, lifecycle, connectors.
workflow[]Ordered step list. Each item is a bare agent name string, a full step dict {"agent": …, "retries": …, "when": …}, or a parallel group {"parallel": […]}.
camelCase keysThe config dict uses camelCase (systemPrompt, mcpUrl) — the same format as the JSON DSL — so configs are portable between Python and TypeScript consumers.
AdditiveWorks alongside the code-first workflow() / parallel() API. Use whichever fits. Custom transform callables still require the code API.

Config schema

The config is a plain dict. Its shape is validated by the JSON Schema at packages/workflow-config.schema.json. Since it's serialisable, you can load it from a file: 

import json
from corti_agent_sdk import from_config

with open("my-workflow.json") as f:
    config = json.load(f)

wf = await from_config(config, agents_client)
result = await wf.run("Patient note…")
KeyRequiredDescription
nameYesUnique name used to reference the agent in the workflow list.
descriptionYesHuman-readable description of the agent's purpose.
systemPromptNoInstructions defining the agent's behaviour. (camelCase — translated to system_prompt internally.)
lifecycleNo"ephemeral" (default) or "persistent".
connectorsNoList of connector dicts. Use camelCase keys: mcpUrl, authType, agentId.

When predicates

The "when" field accepts a declarative predicate dict. If the condition is false, the step is skipped and the previous response passes forward unchanged.

PredicateRuns the step when…
{"text": {"includes": "urgent"}}The previous response text contains "urgent".
{"text": {"notIncludes": "routine"}}The previous response text does not contain "routine".
{"status": "completed"}The previous step's status equals the given TaskState string.
 
# Only escalate when the classifier said "urgent"
{
    "agent": "escalator",
    "when": {"text": {"includes": "urgent"}},
    "retries": 2,
    "retryDelay": 500,
}
Note: retryDelay is in milliseconds in the config (matching the TypeScript convention). from_config() translates it to retry_delay (seconds) automatically when building the Python WorkflowStep.

Parallel groups

The {"parallel": [...]} step fans out to multiple agents concurrently. Fulfilled results are joined and passed to the next step. 

# Fan out to two agents in parallel
{"parallel": ["differential", "redflags"]}

# Override the input for one branch
{"parallel": [
    {"agent": "differential", "input": "Focus on cardiac differentials only."},
    "redflags",
]}

Full code

Source: examples/py/08-workflow-config.py

"""
08 — JSON DSL workflow config.

Define the full agent topology as a plain dict (or load it from a .json file),
then hand it to ``from_config()`` which creates all agents concurrently and
returns a ready-to-run ``Workflow``.

This is an alternative to the code-first ``workflow()`` API — both can be
used in the same project.
"""
import asyncio
import os

from corti_agent_sdk import AgentsClient, from_config
from corti_agent_sdk.client import CortiClient

CONFIG = {
    "agents": [
        {
            "name": "wfc-summarizer",
            "description": "Summarises a clinical note in one sentence.",
            "systemPrompt": "Summarise the note in a single sentence.",
        },
        {
            "name": "wfc-classifier",
            "description": "Classifies a summary as 'urgent' or 'routine'.",
            "systemPrompt": "Reply with exactly one word: 'urgent' or 'routine'. No punctuation.",
        },
        {
            "name": "wfc-differential",
            "description": "Lists a differential diagnosis.",
            "systemPrompt": "List the top 3 differential diagnoses, most likely first. One line each.",
        },
        {
            "name": "wfc-redflags",
            "description": "Flags symptoms warranting urgent evaluation.",
            "systemPrompt": "List any red-flag features from the presentation. One line each.",
        },
        {
            "name": "wfc-escalator",
            "description": "Drafts an escalation for urgent cases.",
            "systemPrompt": "Draft a one-line escalation to the on-call physician.",
        },
    ],
    "workflow": [
        # Step 1: summarise the note
        "wfc-summarizer",
        # Step 2: classify as urgent or routine
        "wfc-classifier",
        # Step 3: fan out differential + red-flags in parallel
        {"parallel": ["wfc-differential", "wfc-redflags"]},
        # Step 4: escalate only when classifier flagged as urgent
        {
            "agent": "wfc-escalator",
            "when": {"text": {"includes": "urgent"}},
            "retries": 2,
            "retryDelay": 500,
        },
    ],
}


async def main() -> None:
    async with CortiClient(
        tenant_name=os.environ["CORTI_TENANT"],
        environment=os.environ.get("CORTI_ENV", "eu"),
        auth={
            "client_id": os.environ["CORTI_CLIENT_ID"],
            "client_secret": os.environ["CORTI_CLIENT_SECRET"],
        },
    ) as client:
        agents = AgentsClient(client)

        wf = await from_config(CONFIG, agents)

        note = "Patient reports severe chest pain radiating to left arm, onset 30 minutes ago."

        result = await wf.run(note)

        print("Final output:", result.output.text)
        print("Steps executed:", len(result.steps))
        print("Stopped early:", result.stopped_early)


asyncio.run(main())

What to expect

Final output: Immediate escalation to on-call physician — patient presenting with acute chest pain radiating to left arm, high suspicion for STEMI. Steps executed: 4 Stopped early: False

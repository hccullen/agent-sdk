03 — Linear pipeline

The simplest shape stateGraph() covers: a chain of agents where each step reads state set by the previous one, with an optional branch that skips escalation for routine cases and automatic retries on failure. 

// Diagram
//
//   summarise ──► classify ──(urgent?)──► escalate ──► END
//                                │
//                                └──► END (routine)

Run it

npm run pipeline

Walkthrough

1 · Define the state

interface PipelineState {
  note:       string;   // the original clinical note
  summary:    string;   // set by summariser
  severity:   string;   // "urgent" | "routine"
  escalation: string;   // set by escalator (or empty for routine cases)
}

2 · Build the graph

const graph = stateGraph<PipelineState>()
  .addNode("summarise", agentNode(
    summarizer,
    (s) => s.note,
    (r, s) => ({ ...s, summary: r.text ?? "" }),
  ))
  .addNode("classify", agentNode(
    classifier,
    (s) => s.summary,
    (r, s) => ({ ...s, severity: (r.text ?? "").trim().toLowerCase() }),
  ))
  .addNode("escalate", agentNode(
    escalator,
    (s) => s.note,
    (r, s) => ({ ...s, escalation: r.text ?? "" }),
    { retries: 2, retryDelay: 500 },
  ))
  .addEdge("summarise", "classify")
  .addEdge("classify", (s) => s.severity.includes("urgent") ? "escalate" : END)
  .addEdge("escalate", END);
No new primitive. A linear pipeline is just a graph with one edge per node. Conditional branches drop in as a routing function on the branching edge; skipping a step is routing straight to END (or to whatever comes after the skipped step).

Retries

The fourth argument to agentNode() is an options object. Set retries to re-invoke the runnable when it returns status === "failed"; retryDelay is the pause between attempts (milliseconds, default 1000). 

agentNode(escalator, getInput, merge, { retries: 2, retryDelay: 500 })

Retries fire only on "failed" — not on "input-required" or "auth-required". If all attempts fail, the final failed response is what gets merged into state.
